アルゴリズミック哲学は、計算美学のムーブメントであり、その後コードで表現されます。.mdファイル（哲学）、.htmlファイル（インタラクティブビューア）、.jsファイル（生成アルゴリズム）を出力します。

これは2ステップで行われます：
1. アルゴリズミック哲学の作成（.mdファイル）
2. p5.js生成アートとして表現（.html + .jsファイル）

最初に、次のタスクを実行します：

## アルゴリズミック哲学の作成

はじめに、以下を通じて解釈される**アルゴリズミック哲学**（静的画像やテンプレートではない）を作成します：
- 計算過程、創発的振る舞い、数学的美
- シード付き乱数、ノイズフィールド、オーガニックシステム
- 粒子、流れ、場、力
- パラメトリックな変動と制御されたカオス

### 重要な理解
- 受け取るもの：ユーザーからの微妙な入力や指示（考慮するが基礎として扱い、創造の自由を縛らない）
- 作るもの：アルゴリズミック哲学／生成美学ムーブメント
- 次に起きること：同じバージョンが哲学を受け取り、**コードで表現**します（p5.jsスケッチは90%がアルゴリズム生成、10%が本質的パラメータ）

このアプローチを検討します：
- 生成アートムーブメントのマニフェストを書く
- 次のフェーズで、それを生かすアルゴリズムを書く

哲学は以下を強調する必要があります：アルゴリズミック表現。創発的振る舞い。計算美。シードによる変動。

### アルゴリズミック哲学の生成方法

**ムーブメント名**（1-2語）：「Organic Turbulence」／「Quantum Harmonics」／「Emergent Stillness」

**哲学の言語化**（4-6段落。簡潔だが完全に）：

アルゴリズミック本質を捉えるために、この哲学が以下を通じてどう現れるかを表現します：
- 計算過程と数学的関係は？
- ノイズ関数と乱数パターンは？
- 粒子挙動と場のダイナミクスは？
- 時間発展とシステム状態は？
- パラメトリック変動と創発的複雑性は？

**重要ガイドライン：**
- **冗長性を避ける**：各アルゴリズミック要素は1回ずつ言及し、ノイズ理論、粒子ダイナミクス、数学原理の繰り返しを避ける（新しい深みがある場合を除く）。
- **職人性を繰り返し強調**：最終アルゴリズムが、膨大な時間をかけて開発され、丁寧に洗練され、その分野の頂点にいる人物の成果に見える必要があることを、哲学の中で複数回強調する。「meticulously crafted algorithm」「the product of deep computational expertise」「painstaking optimization」「master-level implementation」などの表現を繰り返す。
- **創造余地を残す**：アルゴリズミック方向性は具体的に示しつつ、次のClaudeが最高レベルの職人性で解釈・実装できる余白を残す。

哲学は、次のバージョンに「静的画像ではなくアルゴリズムで」アイデアを表現させる必要があります。美は最終フレームではなく、プロセスに宿ります。

### 哲学例

**「Organic Turbulence」**
哲学：自然法則に拘束されたカオス、無秩序から立ち上がる秩序。
アルゴリズミック表現：多層パーリンノイズで駆動されるフローフィールド。ベクトル力に従う数千の粒子が軌跡を蓄積し、有機的な密度マップになる。ノイズの複数オクターブが乱流領域と静穏領域を作る。色は速度と密度から現れる—速い粒子は明るく燃え、遅い粒子は影へ溶ける。アルゴリズムは平衡に至るまで走り、あらゆるパラメータが無数の反復で磨かれた、計算美学の巨匠による緻密な調律の産物になる。

**「Quantum Harmonics」**
哲学：離散的実体が波の干渉パターンを示す。
アルゴリズミック表現：位相値を持つ粒子を格子上に初期化し、サイン波で位相を更新。近接時に位相が干渉し、構成的干渉は明るい結節を、破壊的干渉は空白を生む。単純な調和運動から複雑な曼荼羅が創発する。比率が共鳴美を生むように慎重に選ばれた、執拗な周波数校正の成果。

**「Recursive Whispers」**
哲学：スケールをまたぐ自己相似、有限空間に無限の深さ。
アルゴリズミック表現：再帰的に分岐・細分化する構造。各枝はわずかに乱されつつ黄金比に拘束される。L-systemまたは再帰分割で、数学的でも有機的でもある樹形を生成。微小なノイズ摂動が完全対称を破り、線幅は再帰レベルとともに減衰。分岐角は深い数学探索の産物。

**「Field Dynamics」**
哲学：不可視の力を、その物質への影響で可視化する。
アルゴリズミック表現：数学関数やノイズから構成するベクトル場。端から生まれた粒子が場の線に沿って流れ、平衡や境界で死ぬ。複数場が引力・斥力・回転で粒子に作用する。見えるのは痕跡のみ—不可視の力の幽霊のような証拠。力の釣り合いで綿密に振り付けられた計算のダンス。

**「Stochastic Crystallization」**
哲学：ランダム過程が秩序構造へ結晶化する。
アルゴリズミック表現：ランダム円充填やボロノイ分割。ランダム点から始め、緩和アルゴリズムで進化させる。セルは反発し平衡へ。色はセルサイズ、隣接数、中心距離などで決める。生まれるタイルは偶然にも必然にも見える。各シードが固有の結晶美を生み、巨匠級生成アルゴリズムの印となる。

*上記は凝縮例です。実際のアルゴリズミック哲学は4-6段落で、より実質的な内容にしてください。*

### 必須原則
- **アルゴリズミック哲学**：コードで表現される計算的世界観を作る
- **成果よりプロセス**：美はアルゴリズム実行から生まれ、各実行はユニークであることを常に強調
- **パラメトリック表現**：アイデアは数学関係・力・挙動で伝える（静的構成ではない）
- **芸術的自由**：次のClaudeがアルゴリズム的に解釈できる余地を与える
- **純粋な生成アート**：ランダム付き静止画ではなく「生きたアルゴリズム」を作る
- **熟練職人性**：最終アルゴリズムが、丹念に作り込まれ、無数の反復で磨かれ、計算美学の頂点にいる人物の深い専門性の産物に見える必要があることを繰り返し強調

**アルゴリズミック哲学は4-6段落であるべきです。** 意図されたビジョンを統合する詩的な計算哲学で満たしてください。同じ点の繰り返しを避けてください。このアルゴリズミック哲学を.mdファイルとして出力します。

### 技術要件

**シード付き乱数（Art Blocksパターン）**:
```javascript
// ALWAYS use a seed for reproducibility
let seed = 12345; // or hash from user input
randomSeed(seed);
noiseSeed(seed);
```

**パラメータ構造 - 哲学に従う**:

アルゴリズミック哲学から自然に現れるパラメータを設定するために、「このシステムのどんな性質が調整可能か？」を考えます。

```javascript
let params = {
  seed: 12345,  // Always include seed for reproducibility
  // colors
  // Add parameters that control YOUR algorithm:
  // - Quantities (how many?)
  // - Scales (how big? how fast?)
  // - Probabilities (how likely?)
  // - Ratios (what proportions?)
  // - Angles (what direction?)
  // - Thresholds (when does behavior change?)
};
```

**「パターン種類」ではなく、システムが調整可能であるべき性質に注目して有効なパラメータを設計します。**

**コアアルゴリズム - 哲学を表現する**:

**重要**：何を作るかはアルゴリズミック哲学が決めます。

コードで哲学を表現するには、「どのパターンを使うか？」ではなく「この哲学をどうコードで表現するか？」を考えます。

哲学が**有機的創発**なら：
- 時間とともに蓄積/成長する要素
- 自然規則で拘束されたランダム過程
- フィードバックループと相互作用

哲学が**数学的美**なら：
- 幾何学関係と比率
- 三角関数と調和
- 精密計算から生まれる意外なパターン

哲学が**制御されたカオス**なら：
- 厳密な境界内のランダム変動
- 分岐と相転移
- 無秩序から生まれる秩序

**アルゴリズムはメニューではなく哲学から流れます。**

実装を導くために、概念本質を創造的でオリジナルな選択に反映させます。このリクエスト固有のビジョンを表現するものを構築します。

**キャンバスセットアップ**（標準p5.js構造）:
```javascript
function setup() {
  createCanvas(1200, 1200);
  // Initialize your system
}

function draw() {
  // Your generative algorithm
  // Can be static (noLoop) or animated
}
```

### 職人性要件

**重要**：巨匠生成アーティストが無数の反復から到達したかのように感じるアルゴリズムを作ります。各パラメータを慎重に調律し、あらゆるパターンが目的を持って生まれるようにします。これはランダムノイズではなく、深い専門性で洗練された**制御されたカオス**です。

- **バランス**：視覚ノイズ無しの複雑さ、硬直無しの秩序
- **色の調和**：適当にRGBを振らない。思慮あるパレット
- **構図**：ランダムでも視覚階層と流れを保つ
- **性能**：アニメなら滑らかに動き、最適化する
- **再現性**：同じシードは常に同じ出力

## インタラクティブ成果物の作成

**リマインダー：** `templates/viewer.html`は既に読まれているべきです（ステップ0参照）。そのファイルを開始点として使用します。

生成アートを探索できるよう、単一で自己完結のHTML成果物を作成します。claude.aiや任意のブラウザで即動作する必要があります。すべてをインラインに埋め込みます。

### 重要：固定部分と可変部分

`templates/viewer.html`が基盤です。必要な構造とスタイルが含まれます。

**固定（常にそのまま含める）：**
- レイアウト構造（ヘッダー、サイドバー、メインキャンバス領域）
- Anthropicブランディング（UI色、フォント、グラデーション）
- サイドバーのSeedセクション：
  - Seed表示
  - 前/次ボタン
  - ランダムボタン
  - Seedジャンプ入力 + Goボタン
- サイドバーのActionsセクション：
  - Regenerateボタン
  - Resetボタン

**可変（作品ごとにカスタム）：**
- p5.jsアルゴリズム全体（setup/draw/classes）
- paramsオブジェクト（作品に必要なものを定義）
- サイドバーのParametersセクション：
  - パラメータ制御数
  - パラメータ名
  - スライダーのmin/max/step
  - 制御タイプ（スライダー、入力等）
- Colorsセクション（任意）：
  - 色ピッカーが必要な場合
  - 固定色でもよい
  - モノクロなら不要

**各作品は固有のパラメータとアルゴリズムを持つべきです。** 固定部分は一貫UXを提供し、可変部分が固有ビジョンを表現します。

### 必須機能

**1. パラメータコントロール**
- 数値パラメータ用スライダー（粒子数、ノイズスケール、速度など）
- パレット色用の色ピッカー
- パラメータ変更時のリアルタイム更新
- デフォルトに戻すResetボタン

**2. シードナビゲーション**
- 現在のシード番号表示
- 前/次ボタンでシードを巡回
- ランダムシードボタン
- 特定シードへジャンプする入力
- 要求があれば100バリエーション生成（シード1-100）

**3. 単一成果物の構造**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js from CDN - always available -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* All styling inline - clean, minimal */
    /* Canvas on top, controls below */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- All parameter controls -->
  </div>
  <script>
    // ALL p5.js code inline here
    // Parameter objects, classes, functions
    // setup() and draw()
    // UI handlers
    // Everything self-contained
  </script>
</body>
</html>
```

**重要**：これは単一成果物です。外部ファイル、importは無し（p5.js CDN以外）。すべてインラインです。

**4. 実装詳細 - サイドバー構築**

サイドバー構造：

**1. Seed（固定）** - 必ずそのまま含める：
- Seed表示
- 前/次/ランダム/ジャンプ

**2. Parameters（可変）** - 作品用の制御を作る：
```html
<div class="control-group">
    <label>Parameter Name</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
パラメータ数だけcontrol-groupを追加します。

**3. Colors（任意）** - 必要なら色ピッカーを追加：
- ユーザーがパレットを制御すべきなら追加
- 固定色なら省略可
- モノクロなら省略

**4. Actions（固定）** - 必ずそのまま含める：
- Regenerate
- Reset
- Download PNG

**要件**：
- シード操作が動く（前/次/ランダム/ジャンプ/表示）
- 全パラメータにUIがある
- Regenerate/Reset/Downloadが動く
- Anthropicブランディングを維持（UIスタイルであり、作品の色ではない）

### 成果物の使用

HTML成果物は即動作：
1. **claude.ai**: インタラクティブ成果物として表示され即実行
2. **ファイルとして**: 保存して任意ブラウザで開くだけ（サーバ不要）
3. **共有**: HTMLファイルを送れば完結

  - アルゴリズムはHTML内にインラインで埋め込む（別の.jsファイルは作らない）

**重要リマインダー**：
- テンプレートは「インスピレーション」ではなく**開始点**
- 創造は**アルゴリズム**で行う
- フローフィールド例をコピーしない—哲学が要求するものを作る
- ただしUI構造とAnthropicブランディングはテンプレート通りに維持する
