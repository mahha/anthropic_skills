# ワークフローパターン

## 逐次（シーケンシャル）ワークフロー

複雑なタスクでは、操作を明確で逐次的なステップに分割します。SKILL.mdの冒頭付近でプロセス概要を示すと役立つことが多いです：

```markdown
PDFフォームへの記入は次のステップを含みます：

1. フォームを分析（analyze_form.pyを実行）
2. フィールド対応表を作成（fields.jsonを編集）
3. 対応表を検証（validate_fields.pyを実行）
4. フォームに記入（fill_form.pyを実行）
5. 出力を検証（verify_output.pyを実行）
```

## 条件分岐（コンディショナル）ワークフロー

分岐ロジックがあるタスクでは、判断ポイントを通じてClaudeをガイドします：

```markdown
1. 変更タイプを決定：
   **新規作成？** → 下の「作成ワークフロー」に従う
   **既存編集？** → 下の「編集ワークフロー」に従う

2. 作成ワークフロー: [手順]
3. 編集ワークフロー: [手順]
```
